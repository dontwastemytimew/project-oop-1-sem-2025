Match++: Desktop Dating Agency

Кросплатформний настільний додаток, створений на C++ та фреймворку Qt 6, призначений для симуляції функціоналу сучасної платформи знайомств. Проєкт зосереджений на надійному керуванні даними, ефективних алгоритмах зіставлення та професійній архітектурі.

Архітектура та Технології

Компонент Технологія
Фреймворк Qt 6 (Widgets, Qt Charts)
Мова C++17
Система збірки CMake
База даних SQLite (через QSqlDatabase)
Тестування Google Test & Google Benchmark
Структура Core / Models / GUI / Utils

Унікальна Стилізація Інтерфейсу (IDE Theme)

Головною особливістю UX/UI проєкту є стилізація під середовище розробки Visual Studio Code (Dark+) та класичний C++ код. Це підкреслює аудиторію та назву додатку **(Match++)**.

Основні Функції

User Experience (UX)

  • Інтерфейс одного вікна: Навігація між сторінками (Профіль, Пошук, Метчі) відбувається без перезавантаження, за допомогою QStackedWidget.
  • Сесія: Надійне керування сесією користувача через QSettings.
  • Теми: Перемикання світлої/темної теми за допомогою QSS.

Пошук та Зіставлення

  • Розширений Фільтр: Пошук за віком, містом, статтю та орієнтацією.
  • Автодоповнення Міст: "Розумне" введення міст за допомогою QCompleter, що використовує наявні дані з БД.
  • Картка Профілю: Динамічне відображення даних та відсотка сумісності.
  • Бальна Система: Реалізований алгоритм MatchEngine::compatibilityScore для оцінки сумісності на основі віку та міста.
  • Simulated Chat: Можливість клікнути на метч і відкрити вікно чату із заглушкою та випадковими відповідями бота.

Додаткові фічі

  • 1. Панель Адміністратора:
    • Повний CRUD-інтерфейс для керування користувачами.
    • Генерація 10,000+ фейкових профілів для тестування навантаження.
    • Аналітика (Qt Charts): Візуалізація статистики (розподіл за віком, містами, статтю).
  • 2. Інтернаціоналізація (i18n):
    • Повна підтримка Української та Англійської мов через QTranslator та Qt Linguist.
  • 3. Модульне Тестування (Unit Tests):
    • Тести для DatabaseManager та MatchEngine (використовується Google Test).
  • 4. Логер (UserLogger):
    • Кастомний клас для фіксації критичних подій, помилок та дій користувача.
  • 5. Документація:
    • Повна документація коду (Doxygen) з автоматичним розгортанням на GitHub Pages.

Аналіз Продуктивності (Benchmarks)

Для виявлення вузьких місць було проведено навантажувальне тестування за допомогою бібліотеки Google Benchmark.

Ключові результати (CPU: 3.3 GHz)

Операція Час виконання Висновок та Оптимізація
Логіка матчингу (Math) 15.6 ns Алгоритм розрахунку сумісності надзвичайно швидкий і не потребує оптимізації.
Вставка (Single Insert) 921 µs (0.9 ms) Повільно через відкриття транзакції на кожен запис.
Вставка (Bulk Insert) 3.4 µs / item Прискорення в 270 разів завдяки використанню однієї транзакції. Використовується для генерації даних.
Пошук та Фільтрація (SQL) 230 - 500 ms Основне навантаження. Складні SQL-запити займають до 0.5с, що підтверджує необхідність використання асинхронності в майбутньому.
Сортування (5000 юзерів) 4.5 ms Сортування на стороні C++ працює швидко, пагінація на стороні клієнта допустима.
Копіювання об'єктів 16 ns Механізм Implicit Sharing у Qt робить передачу об'єктів UserProfile майже "безкоштовною" для пам'яті.

Як Запустити Проєкт (CLion / CMake)

  1. Клонування репозиторію: bash git clone [посилання на репозиторій] cd project-oop-1-sem-2025
  2. Завантаження залежностей: Проєкт використовує FetchContent для автоматичного завантаження Google Test та Google Benchmark.
  3. Збірка: Використовуйте стандартний процес збірки CMake (CLion, VS Code або термінал): bash cmake -S . -B build cmake --build build
  4. Запуск Бенчмарків: Щоб відтворити результати тестування продуктивності: bash cmake --build build --target project_benchmark ./build/project_benchmark.exe
